12 ## Question 1 ##
13 
14 We implement the shortest-seek-time-first as well as the circular scan algorithms for disk scheduling.
15 
16 In SSTF.java, we insert requests in sorted ascending order into a doubly linked list. Once a request has been serviced, we see if the request previous to the current one (current.prev) is closer to the current than the one proceeding it (current.next). We update our current head position accordingly.
17 
18 In C-SCAN.java, we maintain one direction for servicing requests, and loop back to the first block once we reach the last block of the disk.
19 
20 We found SSTF to be significantly faster than the provided FCFS algorithm. C-SCAN is the most fair algorithm.
21 
22 In case you want to see our CLOOK, it is located in `DiskCLOOK.java`.
23 ## Question 2 ##
24 
25 In FileSystem.java, we implemented a simple filesystem with 8 inodes, 192 data blocks, and a corresponding 192-bit bitmap.
26 
27 _free\_blocks()_ accesses the bitmap and update values when deleting a file. We generate the bitmask and use the bitwise AND operation to update the bitmap.
28 
29 For writing, we have written a helper method _fill\_blocks()_ which is called by _get\_free\_blocks()_ to update the bitmap for writing. We generate the bitmask and use the bitwise INCLUSIVE OR operation to update the bitmap.
30 
31 A major chunk of the work in reading, writing and deleting is calculating the address on disk using the different offsets. See the code for more details.
